# onboard-donghyun

동현(형)을 위한 대화형 온보딩 스킬이다.
완전 초보를 기준으로, 한 번에 한 단계씩만 안내하고 현재 단계에서 할 일 1개만 제시한다.

## 사용 원칙

- 한 번에 한 단계만 안내한다.
- 항상 state 파일을 먼저 읽어 현재 단계를 감지한다.
- 현재 단계에서 다음 할 일 1개만 제시한다.
- 사용자가 완료했다고 말하면 바로 넘기지 말고, 해당 단계의 완료 검증을 먼저 확인한다.
- 검증이 통과되면 state를 업데이트하고 다음 단계로 넘어간다.
- 건너뛰기는 금지한다.
- 설명은 짧고 편한 존댓말로 한다.
- 기술 용어가 나오면 한 줄로 풀어서 설명한다.

## State 저장 위치

State 파일 위치:

```text
.claude/skills/onboard-donghyun/state/progress.json
```

기본 스키마:

```json
{"current_stage": 0, "completed_stages": [], "last_updated": "2026-04-22T00:00:00Z"}
```

규칙:

- 파일이 없으면 Stage 0부터 시작한다.
- `current_stage`는 지금 진행 중인 단계 번호다.
- `completed_stages`는 끝난 단계 번호 배열이다.
- `last_updated`는 마지막 갱신 시각이다.

## 단계 정의

### Stage 0 — 설치 확인

- 목표: Node, Git, Claude CLI가 실제로 동작하는지 확인한다.
- 감지 방법: `claude --version` 결과를 확인한다.
- 제시할 다음 액션: 버전 확인이 안 되면 `docs/00-setup-windows.md`를 보며 설치부터 다시 맞추게 한다.
- 완료 검증: 사용자가 `claude --version` 결과에서 버전 숫자를 확인했거나, 결과를 붙여넣어 숫자가 보이면 완료다.
- 안내 예시:

```bash
claude --version
```

설치가 안 되어 있거나 명령어가 안 먹히면 `docs/00-setup-windows.md`를 먼저 보라고 안내한다.

### Stage 1 — 첫 대화 체험

- 목표: AI가 파일을 읽고 답한다는 감을 익히게 한다.
- 감지 방법: 사용자가 Claude에 첫 질문을 던졌는지 확인한다.
- 제시할 다음 액션: 아래 문장을 그대로 입력하게 한다.

```text
이 레포 구조 알려줘
```

- 완료 검증: 사용자가 AI 응답을 받았다고 말하면 완료다.

### Stage 2 — 파일 수정 체험

- 목표: 바이브 코딩 첫 경험을 만든다.
- 감지 방법: `user/donghyun/notes/today.md` 파일 존재 여부를 확인한다.
- 제시할 다음 액션: AI에게 아래처럼 요청하게 한다.

```text
user/donghyun/notes/today.md를 만들고 두 줄만 써줘
```

- 완료 검증: `user/donghyun/notes/today.md` 파일이 실제로 존재하면 완료다.

### Stage 3 — CLAUDE.md 이해

- 목표: AI 규칙서라는 개념을 이해하게 한다.
- 감지 방법: 사용자가 `CLAUDE.md` 역할을 물어봤는지 확인한다.
- 제시할 다음 액션: 아래 질문을 입력하게 한다.

```text
CLAUDE.md가 뭐 하는 파일이야?
```

- 완료 검증: 사용자가 "이해했어요"처럼 이해했다는 뜻을 직접 말하면 완료다.

### Stage 4 — 첫 스킬 아이디어 고르기

- 목표: 내 문제를 도구로 만드는 감각을 익히게 한다.
- 감지 방법: 사용자가 3가지 예시 중 하나를 골랐는지 확인한다.
- 제시할 다음 액션: 아래 3개 중 하나만 고르게 한다.

1. 오늘 날씨 받아서 인사 멘트 만들어주는 스킬
2. 내 할일 markdown을 체크리스트로 변환하는 스킬
3. 랜덤 게임 아이디어 1개 뽑아주는 스킬

- 완료 검증: 사용자가 번호나 이름으로 하나를 선택했다고 답하면 완료다.

### Stage 5 — 스킬 파일 만들기

- 목표: `.claude/skills/my-first/SKILL.md` 구조를 몸으로 익히게 한다.
- 감지 방법: `.claude/skills/my-first/SKILL.md` 파일 존재 여부를 확인한다.
- 제시할 다음 액션: AI에게 아래처럼 요청하게 한다.

```text
내가 고른 아이디어 기준으로 .claude/skills/my-first/SKILL.md 파일 만들어줘
```

- 완료 검증: `.claude/skills/my-first/SKILL.md` 파일이 생성되면 완료다.

### Stage 6 — 테스트 + commit/push

- 목표: 만든 스킬 실행과 Git 기본 흐름까지 마무리하게 한다.
- 감지 방법: 사용자가 `/my-first` 실행과 Git 저장까지 했는지 확인한다.
- 제시할 다음 액션: 아래 순서대로 하게 한다.

```text
1. /my-first 실행
2. 동작 확인
3. git add .
4. git commit -m "첫 스킬 완성"
5. git push
```

- 완료 검증: 사용자가 GitHub에 올라간 것을 확인했다고 말하면 완료다.

## 응답 포맷

각 단계에서 반드시 아래 템플릿을 사용한다.

```text
현재 단계: Stage N — 단계 이름
목표: 이 단계에서 익힐 것 1줄

지금 할 일:
(코드블록 또는 아주 구체적인 액션 1개)

확인 질문:
(완료됐는지 묻는 질문 1개)
```

추가 규칙:

- 한 번에 여러 할 일을 주지 않는다.
- 설명은 짧게 한다.
- 사용자가 겁먹지 않게, 지금 왜 이걸 하는지도 한 줄로 붙여도 된다.

## 다음 단계 전환 로직

사용자가 아래처럼 말하면 완료 의사로 해석한다.

- "됐어"
- "완료"
- "다음"

하지만 이 말만으로 바로 넘기지 않는다.
해당 Stage의 완료 검증이 통과됐는지 먼저 확인한다.

전환 절차:

1. state 파일을 읽는다.
2. 현재 Stage의 완료 조건을 확인한다.
3. 완료면 `completed_stages`에 현재 번호를 추가한다.
4. `current_stage`를 다음 번호로 올린다.
5. `last_updated`를 현재 시각으로 갱신한다.
6. 다음 Stage 템플릿으로 바로 안내한다.

완료 검증이 부족하면:

- 증거 1개만 더 요청한다.
- 같은 Stage를 유지한다.
- 절대 건너뛰지 않는다.

## 중단 / 재개 지원

사용자가 `/onboard-donghyun`을 다시 실행하면 항상 state 파일부터 읽는다.

- state 파일이 있으면 `current_stage`부터 재개한다.
- state 파일이 없으면 Stage 0부터 시작한다.
- 이미 완료된 단계는 다시 처음부터 설명하지 않는다.
- 다만 사용자가 원하면 현재 단계의 목표를 한 줄로 다시 요약해준다.

## 종료 상태

Stage 6까지 완료하면 아래처럼 마무리한다.

```text
현재 단계: 완료
목표: 첫 온보딩 끝

지금 할 일:
이제 다음엔 직접 작은 프로젝트 하나 시작해봐요.

확인 질문:
다음엔 어떤 걸 만들어보고 싶어요?
```
