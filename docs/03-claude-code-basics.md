# 03. Claude Code 기본기

처음 `claude` 실행한 후 뭐부터 할지. 바이브 코딩이 뭔지 감 잡는 장.

---

## Claude Code가 뭐야?

**터미널에서 돌아가는 AI 개발 동료.**

- 일반 Claude.ai는 웹에서 대화만 함
- Claude Code는 **내 컴퓨터 파일을 읽고, 수정하고, 명령어도 실행**할 수 있음
- 즉, "같이 개발하는 사람"에 가까움

---

## 첫 실행

레포 폴더에서:
```bash
claude
```

화면이 바뀌고 입력창이 떠요. 이제 AI와 대화가 시작돼요.

---

## 첫 대화 연습 5개

순서대로 한 번씩 쳐보세요. 감 잡힐 거예요.

### 1. 읽기 시켜보기
```
이 레포 구조 알려줘
```
→ AI가 파일 트리를 훑고 설명해줘요. **"AI가 내 파일을 실제로 읽는다"**는 감각.

### 2. 질문하기
```
CLAUDE.md가 뭐 하는 파일이야?
```
→ 이 레포의 규칙서를 읽고 요약해줘요.

### 3. 파일 만들어 달라 하기
```
user/donghyun/notes/ 안에 today.md를 만들어서 "오늘 배운 것" 이라고만 써줘
```
→ AI가 실제로 파일을 생성해요. 만들어지면 **실제로 존재하는지 확인**해보세요.
```bash
ls user/donghyun/notes/
```

### 4. 수정 시키기
```
방금 만든 today.md에 "Claude Code 재밌다" 한 줄 추가해줘
```

### 5. 이해 안 가는 거 물어보기
```
터미널이 뭐야? 맥이랑 윈도우에서 뭐가 달라?
```
→ AI는 그냥 선생님처럼 설명해주는 것도 잘해요.

## 슬래시 커맨드 (`/`로 시작하는 명령어)

입력창에 `/`만 치면 쓸 수 있는 명령어 리스트가 떠요.

자주 쓸 것들:

| 명령어 | 설명 |
|--------|------|
| `/onboard-donghyun` | **내 대화형 온보딩** (현재 단계 감지 + 다음 할 일) |
| `/help` | 도움말 |
| `/clear` | 대화 내용 초기화 (컨텍스트 리셋) |
| `/compact` | 대화가 너무 길어졌을 때 요약 |
| `/model` | 어떤 AI 모델 쓸지 바꾸기 |
| `/exit` | Claude Code 종료 |

> **Tab 누르면 자동완성**. `/onb` 까지 치고 Tab 누르면 나머지 채워줘요.

## 특수 키

| 키 | 하는 일 |
|----|---------|
| **Ctrl+C** | 지금 AI가 하는 작업 즉시 중단 |
| **Ctrl+D** | Claude Code 종료 |
| **Esc** | 입력 중인 메시지 취소 |
| **↑ / ↓** | 이전에 쳤던 명령 불러오기 |
| **Shift+Enter** | 줄바꿈 (엔터는 전송) |

> **AI가 이상한 짓 하고 있으면 망설이지 말고 Ctrl+C**. 언제든 멈출 수 있어요.

## CLAUDE.md vs CLAUDE.local.md

둘 다 AI가 자동으로 읽는 "규칙서"예요.

| 파일 | 용도 | Git 포함? |
|------|------|----------|
| `CLAUDE.md` | 이 레포 **공용 규칙** (AI 접근권한, 말투, 사용자 구분) | ✅ 올라감 |
| `CLAUDE.local.md` | **나만의 개인 선호** (말투, 응답 형식, 관심사) | ❌ 제외 |

> **처음에는 `CLAUDE.local.md` 안 만들어도 돼요.** 나중에 "AI가 내 스타일에 안 맞네" 싶을 때:
> ```bash
> cp CLAUDE.local.md.template CLAUDE.local.md
> ```
> 복사한 다음 내용 채우면 돼요.

## 바이브 코딩이 뭐야?

"**코드를 완벽히 이해하고 쓰자**" (X)
"**일단 AI에게 만들어 달라 하고, 돌려보고, 궁금한 건 물어보며 배우자**" (O)

### 전통 방식 (피곤함)
1. Python 문법 책 300p 읽기
2. 연습 문제 100개 풀기
3. 3개월 뒤 드디어 뭔가 만들기 시작

### 바이브 코딩
1. "숫자 맞히기 게임 파이썬으로 만들어줘" → AI가 만듦
2. 돌려봄 → 작동함
3. **"이 줄은 뭐 하는 거야?"** 하나씩 물어봄
4. 조금 바꿔봄 → 에러 → AI에게 보여주고 물어봄
5. 어느새 **배우고 있음**

핵심은 **"일단 돌아가는 걸 손에 쥐고, 뜯어보며 이해"**.

---

## 바이브 코딩 3원칙

### 1. 물어보는 걸 부끄러워하지 말 것

AI는 **초보 질문일수록 친절하게** 답해요. "이거 너무 기본 아닌가" 같은 거 없음.

```
"변수가 뭐야?"
"이 괄호는 왜 있어?"
"지금 내가 뭘 하고 있는 거야?"
```

다 괜찮은 질문이에요.

### 2. 한 번에 하나만

큰 걸 한 번에 달라고 하지 말고, **작게 쪼개서**:

❌ "레이싱 게임 만들어줘"
✅ "화면에 자동차 사진 한 장 띄워줘"
→ 됨
✅ "이제 방향키로 움직이게 해줘"
→ 됨
✅ "이제 장애물 하나 추가해줘"

**한 단계씩 쌓으면 이해가 따라오고 디버깅도 쉬워요.**

### 3. 망해도 Git이 되돌려줌

실험 전에 commit만 해두면 어떤 실수를 해도 돌아올 수 있어요:

```bash
git add .
git commit -m "실험 전 안전 지점"
# ... 실험 ...
# 망했으면:
git checkout .
```

**"망하면 어쩌지?" 불안이 사라지면 배움이 빨라져요.**
