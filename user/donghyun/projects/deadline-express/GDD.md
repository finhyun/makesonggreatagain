# 데드라인 익스프레스 (Deadline Express)

**부제:** 흐름(Flow)을 지배하는 자가 황무지를 돌파한다

---

## 1. 기획 배경 및 의도

본 프로젝트는 **"왜 디펜스 게임은 항상 적이 올 때까지 기다려야 하는가?"** 라는 질문에서 시작되었습니다.

기존 디펜스 장르의 정형화된 문법인 **'웨이브(Wave)'** 시스템을 과감히 제거하고, '연속적인 흐름'을 핵심 축으로 삼습니다.

- **웨이브가 없는 이유:** 적군이 무리지어 '오는 것'이 아니라, 플레이어가 적진을 향해 '달려가는' 구조를 만들기 위함
- **기획 의도:** 게임이 정해준 타이밍에 싸우는 것이 아니라, 플레이어가 기차의 속도를 조절함으로써 스스로 교전의 밀도와 난이도를 결정하게 만드는 **'자율적 템포 디펜스'** 지향

---

## 2. 개요 (Vision)

| 항목 | 내용 |
|------|------|
| 장르 | 2D 사이드뷰 이동형 디펜스 (Continuous Flow Defense) |
| 시점 | 2D Side-View (기차 단면을 옆에서 보는 방식) |
| 핵심 컨셉 | 멈추지 않는 황무지 질주 속, 엔진과 화력을 동시에 관리하는 요새 운영 |
| 승리 조건 | 기관차가 파괴되지 않고 최종 목적지에 도착 |
| 패배 조건 | 기관차 HP 0 또는 연료 고갈 |

**로그라인:**
> "설국열차의 생존과 매드맥스의 질주가 만난 '무박자' 디펜스 — 1대의 기관차와 3종의 모듈로 구성된 요새를 이끌고, 빨리 달리면 엔진이 터지고 천천히 가면 목숨이 위태로운 극한의 딜레마를 뚫고 나가라."

---

## 3. 절대 규칙 (The Grand Law)

> **"빨리 달리면 적을 따돌릴 수 있지만 엔진이 타버리고, 천천히 가면 자원을 모을 수 있지만 적에게 포위당한다."**

이 게임의 모든 전략적 판단은 아래 물리 법칙을 따릅니다.

| 상태 | 장점 | 단점 |
|------|------|------|
| 고속 주행 | 적 사격 범위 조기 이탈, 충돌 대미지 유발 | 엔진 과열(화재 발생), 자원 채집 효율 급감 |
| 저속 주행 | 자원 자동 흡입 안정화, 냉각 효율 상승 | 적에게 포위당할 확률 증가 |

- **열(Heat) 관리:** 기차 가동 및 포탑 사격 시 엔진 온도 상승 → 100% 도달 시 무작위 칸 화재 발생
- **속도는 실시간 조절 가능** (전투 중에도 레버 조작 가능)

---

## 4. 기차 기본 구성: 1 + 3 시스템

### A. 머리: 기관차 (The Locomotive) — 고정
- 기차의 맨 앞, 엔진이 위치한 심장부
- 엔진 열 관리, 연료 게이지, 속도 제어 레버 담당
- **파괴 또는 연료 고갈 = 게임 오버**

### B. 몸통: 3가지 기본 모듈 (정비소에서 순서 재배치 가능)

| 모듈 | 역할 |
|------|------|
| 전투 칸 (Combat) | 외부 포탑으로 접근하는 적 자동 요격. 장갑 두껍지만 전력 소모 큼 |
| 유지보수 칸 (Maintenance) | 엔진 냉각 장치 + 수리 드론 사출구. 내부 위기 대응 핵심 |
| 자원채집 칸 (Harvesting) | 자기장으로 주변 고철·연료 자동 흡입. 저속일수록 흡입력 강함 |

---

## 5. 자원 시스템

### 자동 자원 채집
- 채집 모듈 주변 흡입 반경 내 자원이 자동으로 기차로 날아와 저장
- 플레이어는 채집 조작 대신 **속도를 늦추는 전략적 판단**에만 집중

### 자원 종류
| 자원 | 용도 |
|------|------|
| 고철 (Scrap) | 장갑 수리, 포탑 강화 |
| 연료 (Fuel) | 기차 이동 필수 자원, 고갈 시 게임 오버 |
| 냉각수 (Coolant) | 엔진 열 즉시 냉각 소모품 제작 |

---

## 6. 이중 방어 시스템 (Dual Defense)

### 외부 방어
- 기차 밖 약탈자 차량, 바이크, 비행 괴수를 포탑으로 격퇴
- 각 칸의 외벽 장갑 수치 관리 필요

### 내부 대응
장갑이 파괴되어 적이 침입할 경우:

| 대응 수단 | 설명 |
|-----------|------|
| 격리벽 (Bulkhead) | 침입 칸의 앞뒤 문 봉쇄 → 해당 칸 기능 정지 |
| 방어 드론 | 수리 드론을 전투 모드로 전환, 내부 침입자 소탕 |
| 칸 분리 (Eject) | 점령당한 칸을 통째로 분리 탈락 → 기차 길이 단축되지만 생존 가능 |

---

## 7. 월드 맵 & 진행 방식

### 노드 기반 지도
- 시작점 → 최종 목적지까지 여러 갈래 경로 존재
- 플레이어는 기차 상태(연료, 장갑)에 따라 경로 선택

| 루트 | 특징 |
|------|------|
| 지름길 | 거리 짧아 연료 절약 but 적 밀도 높아 엔진 풀가동 필요 |
| 우회로 | 거리 길어 연료 소모 크지만 자원 풍부, 안전 |

### 플레이 호흡

| 단계 | 시간 |
|------|------|
| 노드 플레이 (Action Phase) | 약 8~12분 |
| 정비 구간 (Safe Zone) | 약 2~3분 |
| 전체 런 (Full Run) | 약 60분 내외 |

### 정비 구간 (Safe Zone)
- 노드 끝의 정착지 도착 시 전투 소강 → 자원으로 장갑 수리, 모듈 업그레이드, 칸 순서 재배치

---

## 8. 핵심 재미 요소 (Core Fun)

1. **예측 불가능한 긴장감** — 정해진 웨이브 없이, 내가 달리는 환경 밀도가 난이도를 결정
2. **완급 조절의 묘미** — "자원을 먹기 위해 속도를 줄일 것인가, 엔진을 태우며 돌파할 것인가"
3. **이중 관리의 쾌감** — 밖에서 포탄이 빗발치고 안에서 불꽃이 튀는 극한 상황을 통제했을 때의 성취감
