# Deadline Express MVP 플레이테스트 체크리스트

## 0. 테스트 목적

- 목적: 5분 풀런 3회로 MVP 재미 게이트와 클라이맥스 흐름을 확인한다.
- 기준 스펙: `spec.md` §2 재미 게이트, §10.2 수동 테스트, §11 마일스톤 M5~M7.
- 대상 빌드: `index.html` 단일 HTML 실행본.
- 기록 원칙: 체감이 아니라 화면 타이머 기준으로 적고, 애매하면 실패 쪽으로 표시한다.

## 2. 페이즈 기준

| 페이즈 | 스펙상 구간 | 진입 기준 |
|---|---:|---|
| Warm-up | 0:00-0:45 | 시작 직후 |
| Crescendo | 0:45-2:15 | 타이머 0:45 경과 |
| Pinch | 2:15-3:30 | 타이머 2:15 경과 |
| Climax | 3:30-4:30 | 타이머 3:30 경과, 코뿔 트럭 구간 |
| Sprint | 4:30-5:00 | 타이머 4:30 경과, 결승선 가시화 |

## 3. 이벤트 카드 표기

| 구현 ID | 표시명 | 기록할 내용 |
|---|---|---|
| storm | 모래폭풍 | 발동 시간, 시야/명중 체감 |
| canyon | 협곡 좁힘 | 발동 시간, 사거리/출현 방향 체감 |
| scrap | 고철 더미 | 발동 시간, 자원 풍족감과 적 압박 |
| ghost | 유령 신호 | 발동 시간, 가짜/진짜 적 반응 |
| leak | 냉각수 누수 | 발동 시간, 열 압박 |
| sos | 무전 SOS | 발동 시간, A 수락 또는 B 무시 |

## 4. 재미 게이트 판정법

| 항목 | PASS 기준 | 측정 방법 |
|---|---|---|
| 지루구간 | 아무 입력 없이도 안전한 시간이 30초 이하 | 각 런에서 가장 길었던 안전 방치 시간을 초 단위로 기록 |
| 결정 압박 빈도 | 30초마다 최소 1회 이상 의미 있는 선택 발생 | 속도, 사격, 내부 전환, 격리, 냉각수, SOS, BURNOUT 등 선택 횟수 기록 |
| BURNOUT 위치 | 사용 시점이 Climax에 자연스럽게 위치 | 4:00 이후 해금, Climax/Sprint 전환 체감 기록 |
