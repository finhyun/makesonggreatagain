# favorite-skills — 이것만 알아도 90% 커버돼요

형이 처음부터 많은 걸 알 필요는 없어요.
아래 6개만 익혀도 대부분의 상황은 커버돼요.

---

## 빠른 표

| 스킬 | 언제 써요? | 예시 한 줄 |
|------|------------|-------------|
| `/plan` | 큰 작업 전에 순서 잡고 싶을 때 | `/plan 로그인 화면부터 어디까지 만들지 정리해줘` |
| `/skill` | 설치된 스킬을 보고 싶을 때 | `/skill 지금 뭐가 있는지 보여줘` |
| `/learner` | 자주 하는 패턴을 스킬로 뽑고 싶을 때 | `/learner 방금 한 작업 패턴을 스킬로 정리해줘` |
| `/setup-local` | 내 취향을 Claude에 알려주고 싶을 때 | `/setup-local 답변 짧게, 한국어 존댓말로 기억해줘` |
| `/brainstorming` | 아이디어를 넓게 뽑고 싶을 때 | `/brainstorming 미니 게임 아이디어 10개 줘` |
| `/debugging` | 에러 원인을 좁히고 싶을 때 | `/debugging 이 에러 왜 나는지 단계별로 봐줘` |

---

## `/plan`

저는 큰 작업 전에 거의 먼저 써요.
뭘 만들지 머릿속에 대충만 있을 때, 순서를 쪼개주면 훨씬 덜 막혀요.

형이 이럴 때 쓰면 좋아요.

- 게임 만들고 싶은데 어디서 시작할지 모를 때
- 한 번에 너무 많은 걸 하려 할 때
- 오늘 할 일 1개만 뽑고 싶을 때

---

## `/skill`

지금 쓸 수 있는 스킬 목록을 보는 용도예요.
새로 설치된 게 있는지, 이름이 정확히 뭔지 확인할 때 편해요.

저는 이름이 헷갈릴 때 그냥 이걸 먼저 열어요.

---

## `/learner`

이건 대화나 작업 패턴에서 "이거 자주 반복하네" 싶은 걸 뽑아주는 쪽에 좋아요.
반복 작업을 나중에 버튼처럼 쓰고 싶을 때 감을 잡게 해줘요.

형이 같은 요청을 세 번쯤 반복했다면 한 번 써볼 만해요.

---

## `/setup-local`

내 스타일을 Claude에 맞추는 도구라고 보면 돼요.
말투, 응답 길이, 자주 쓰는 언어 같은 걸 개인 설정으로 남기고 싶을 때 좋아요.

저는 보통 이런 식으로 써요.

```text
/setup-local 한국어 존댓말, 짧게 답하고 모르면 먼저 질문해줘
```

---

## `/brainstorming`

창작 작업 전에는 이게 꽤 좋아요.
아직 구현보다 아이디어 단계일 때, 넓게 뽑고 그중 하나를 고르는 흐름에 잘 맞아요.

예를 들면 이런 데 써요.

- 게임 콘셉트 아이디어 뽑기
- 이름 후보 모으기
- 기능 후보를 거칠게 정리하기

---

## `/debugging`

버그 만났을 때는 감으로 건드리지 말고 이걸 써보면 좋아요.
문제를 단계별로 좁히는 흐름이라 초보일수록 도움이 커요.

특히 아래처럼 주면 좋아요.

```text
/debugging 이 에러 전문은 이거고, 방금 바꾼 파일은 app.js야
```

---

## 제가 쓰는 순서 예시

1. 만들고 싶은 게 생기면 `/brainstorming`
2. 방향이 잡히면 `/plan`
3. 자주 쓰는 방식이 보이면 `/learner`
4. 환경이나 말투를 맞추고 싶으면 `/setup-local`
5. 이름이나 목록이 헷갈리면 `/skill`
6. 막히면 `/debugging`

---

## 한 줄 요약

형은 **`/plan`, `/brainstorming`, `/debugging`** 이 세 개부터 손에 익히면 가장 빨리 체감될 거예요.
