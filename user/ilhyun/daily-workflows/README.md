# daily-workflows — 제가 매일 돌리는 루틴

형이 처음엔 이것만 따라 해도 돼요.
핵심은 **작업 시작 전 pull, 작업 중 status, 끝나면 commit/push** 예요.

---

## 왜 이 순서로 해요?

Git은 작업 세이브 파일이에요.
먼저 최신 내용을 받고(`git pull`), 중간중간 상태를 보고(`git status`), 끝나면 저장하고 올리면(`git commit`, `git push`) 실험적으로 코딩해도 덜 불안해요.

제가 이 순서를 고정하는 이유는 2가지예요.

- 어제 어디까지 했는지 바로 이어가기 쉬워요
- 망해도 마지막 커밋으로 돌아갈 수 있어요

---

## 아침 시작 루틴

### 1. 레포로 들어가요

```bash
cd ~/dev/레포이름
```

### 2. 최신 상태를 먼저 받아요

```bash
git pull
```

> `pull`은 GitHub에 올라간 최신 내용을 내 컴퓨터로 가져오는 명령어예요.

### 3. Claude를 켜요

```bash
claude
```

### 4. 시작 전에 짧게 자가점검해요

저는 보통 Claude에게 이렇게 먼저 말해요.

```text
어제 뭐 하다 말았지? 지금 상태 먼저 정리해줘
```

이 한 줄만 해도 머리가 빨리 돌아와요.

---

## 작업 중 루틴

30분~1시간마다 한 번씩 이걸 봐요.

```bash
git status
```

`status`는 지금 뭐가 바뀌었는지 보여주는 명령어예요.

이럴 때 특히 좋아요.

- 파일을 몇 개 건드렸는지 헷갈릴 때
- 지금 저장 안 된 작업이 얼마나 쌓였는지 보고 싶을 때
- Claude가 어디까지 바꿨는지 감을 잡고 싶을 때

저는 길게 작업하면 중간에 한 번씩 이렇게 물어봐요.

```text
지금까지 바뀐 파일 기준으로 내가 뭘 했는지 3줄로 정리해줘
```

---

## 세션 마감 루틴

작업 끝나면 거의 이 순서 그대로 가요.

```bash
git add .
git commit -m "오늘 한 것"
git push
```

그다음 Claude에서 나와요.

```text
/exit
```

> `add`는 저장 준비, `commit`은 체크포인트 저장, `push`는 GitHub 업로드예요.

커밋 메시지는 완벽할 필요 없어요.
처음엔 `오늘 한 것`, `메모 추가`, `게임 버튼 테스트` 정도로만 써도 충분해요.

---

## 제가 자주 쓰는 하루 타임라인 예시

| 시간대 | 제가 하는 일 |
|--------|--------------|
| 오전 9:00 | `cd ~/dev/레포이름` |
| 오전 9:01 | `git pull` |
| 오전 9:03 | `claude` 실행 |
| 오전 9:05 | "어제 뭐 하다 말았지?" 자가점검 |
| 오전 9:10~11:30 | 작은 작업 1개 진행 |
| 오전 10:00쯤 | `git status` 한 번 확인 |
| 오후 1:30 | 다시 시작 전에 `git status` 또는 Claude에게 상태 요약 요청 |
| 오후 2:00~4:30 | 기능 추가, 메모 정리, 질문하기 |
| 오후 3:00쯤 | `git status` 한 번 더 확인 |
| 저녁 마감 | `git add .` → `git commit -m "오늘 한 것"` → `git push` → `/exit` |

---

## 형 기준 체크리스트

- [ ] 작업 시작 전에 `git pull` 했어요
- [ ] Claude 켜기 전에 레포 위치 맞는지 확인했어요
- [ ] 30분~1시간마다 `git status`로 감을 봤어요
- [ ] 끝나기 전에 commit/push 했어요
- [ ] `/exit`로 세션을 정리했어요
