# 02. Claude의 메모리 시스템 이해

형이 "왜 어떤 건 기억하고 어떤 건 또 물어보지?" 싶을 수 있어요.
Claude 쪽 기억은 보통 **공용 규칙, 개인 선호, 장기 메모** 이 세 층으로 생각하면 편해요.

---

## 3가지 메모리 한눈에 보기

| 이름 | 용도 | 누가 관리해요? | 범위 |
|------|------|----------------|------|
| `CLAUDE.md` | 이 레포에서 지킬 공용 규칙 | 레포 쪽에서 관리 | 레포 공통 |
| `CLAUDE.local.md` | 내 취향, 말투, 작업 습관 | 사용자 본인 | 내 로컬 환경 |
| `MEMORY.md` | 대화하면서 쌓이는 장기 메모 개념 | AI가 자동으로 쌓거나 갱신 | 세션을 넘겨 이어질 수 있음 |

---

## `CLAUDE.md`

이건 레포 공용 규칙서예요.
예를 들면 이런 걸 담아요.

- 어느 폴더를 써도 되는지
- 말투를 어떻게 맞출지
- 초보 기준으로 얼마나 쉽게 설명할지

쉽게 말하면 **"이 레포에서의 기본 약속"** 이에요.

---

## `CLAUDE.local.md`

이건 형 개인 취향 메모예요.
Git에 안 올리고, 내 컴퓨터에서만 쓰는 쪽이라고 보면 돼요.

예를 들면 이런 걸 적어둘 수 있어요.

- 답변은 짧게 해줘
- 한국어 존댓말로 해줘
- 모호하면 먼저 질문해줘
- 내가 Git 초보라는 걸 기준으로 설명해줘

즉, **"형한테 맞춘 개인 설정"** 이에요.

---

## `MEMORY.md` 개념

이건 조금 다르게 보면 돼요.
파일이 실제로 보일 수도 있고, 환경에 따라 안 보일 수도 있어요.
중요한 건 이름보다 개념이에요.

AI가 대화하면서 이런 걸 장기 기억처럼 쌓는다고 생각하면 돼요.

- 내가 좋아하는 답변 방식
- 자주 하는 작업 패턴
- 반복해서 준 피드백

즉, **"계속 대화하면서 생기는 누적 기억"** 쪽이에요.

---

## "AI야 이거 기억해줘" 패턴

형은 그냥 자연어로 말해도 돼요.

예시:

```text
앞으로 Git 설명할 땐 게임 세이브 비유로 설명해줘
```

```text
내가 에러를 보내면 해결책보다 원인 설명부터 먼저 해줘
```

```text
앞으로는 답변을 짧게 주고, 내가 더 물으면 자세히 풀어줘
```

이런 한 줄이 쌓이면 다음 세션부터도 점점 맞아져요.

---

## 피드백 저장은 이렇게 생각하면 돼요

형이 뭔가 불편했을 때 길게 말할 필요 없어요.
이 한 줄이면 충분할 때가 많아요.

```text
앞으로 이렇게 해줘: 먼저 요약 3줄, 그다음 내가 원하면 자세한 설명
```

핵심은 **불만만 말하지 말고 원하는 형식을 같이 말하는 것**이에요.
그러면 다음부터 훨씬 잘 맞아요.

---

## 언제 뭘 건드리면 돼요?

- 레포 전체 규칙을 바꾸고 싶다 → `CLAUDE.md`
- 내 개인 취향을 저장하고 싶다 → `CLAUDE.local.md`
- 대화하면서 스타일을 길게 맞춰가고 싶다 → 메모리 피드백 한 줄

---

## 한 줄 요약

형은 일단 **`CLAUDE.md`는 공용 규칙, `CLAUDE.local.md`는 내 취향, 메모리는 장기 기억** 이렇게만 잡고 가면 충분해요.
