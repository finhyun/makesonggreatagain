# 01. AI와 대화 잘 하는 법

형이 Claude를 잘 쓰는 사람과 덜 잘 쓰는 사람의 차이는, 코딩 실력보다 **질문하는 방식**에서 먼저 갈려요.
어렵게 할 필요 없고, 몇 가지만 기억하면 돼요.

---

## 핵심 5가지

### 1. 모호하면 먼저 질문하게 시켜요

처음부터 이렇게 말해도 돼요.

```text
이해 안 되는 부분이 있으면 바로 추측하지 말고 먼저 물어봐
```

이 한 줄이 중요한 이유는, AI가 애매한 상태에서 멋대로 달리기 시작하는 걸 줄여줘요.

---

### 2. 한 번에 하나만 요청해요

초보일수록 이게 진짜 중요해요.

나쁜 예:
```text
로그인 만들고, 디자인도 예쁘게 하고, 배포도 하고, 에러도 잡아줘
```

좋은 예:
```text
로그인 화면 HTML부터 먼저 만들어줘
```

한 번에 하나씩 가면, 어디서 잘됐고 어디서 꼬였는지 바로 보여요.

---

### 3. 맥락은 짧아도 구체적으로 줘요

AI는 점쟁이가 아니에요.
파일 경로, 에러 전문, 방금 한 행동을 같이 주면 훨씬 정확해져요.

좋은 재료는 이런 거예요.

- 파일 경로: `user/donghyun/projects/game/app.js`
- 에러 전문: 화면에 나온 문장 그대로
- 방금 한 행동: "버튼 코드 추가하고 저장했어요"

특히 에러는 **요약하지 말고 그대로 복붙**하는 게 좋아요.

---

### 4. "왜 그랬어?"를 꼭 물어봐요

돌아가는 코드만 받으면 반은 한 거고, 이유까지 이해하면 그때부터 내 실력이 돼요.

저는 보통 이런 식으로 다시 물어요.

```text
왜 이렇게 바꿨어? 초보 기준으로 한 줄씩 설명해줘
```

이 습관이 있어야 AI가 만든 코드를 내 머리로 가져올 수 있어요.

---

### 5. AI가 헛짓하면 Ctrl+C 하고 다시 가요

이건 진짜 중요한 습관이에요.
AI가 이상한 방향으로 가면 억지로 끝까지 보지 말고 멈추면 돼요.

- `Ctrl+C` — 지금 하는 작업 즉시 중단
- 다시 짧게 재지시 — 원하는 방향만 다시 말하기

예를 들면 이렇게요.

```text
방금 방향은 아니야. app.js만 보고 에러 원인만 찾아줘
```

---

## 바로 써먹는 예시 4개

### 예시 1. 처음 구조 파악

```text
이 레포 구조를 읽고 초보 기준으로 설명해줘. 이해 안 되면 먼저 물어봐
```

### 예시 2. 파일 하나만 수정

```text
user/donghyun/notes/today.md만 보고 두 줄 메모 추가해줘
```

### 예시 3. 에러 디버깅

```text
app.js에서 버튼 누르면 에러가 나요.
에러 전문은 아래예요.
TypeError: Cannot read properties of null (reading 'addEventListener')
왜 그런지 먼저 설명해줘
```

### 예시 4. 코드 이해하기

```text
방금 만든 코드에서 함수 부분만 설명해줘. 왜 이렇게 썼는지도 같이 알려줘
```

---

## 형 기준 대화 템플릿

막막할 때는 아래 틀만 복붙해도 돼요.

```text
내 목표: 
지금 상태: 
관련 파일 경로: 
에러 전문(있으면): 
이해 안 되면 먼저 질문해줘
```

---

## 한 줄 요약

형이 AI를 잘 쓰려면, **크게 시키지 말고 구체적으로 하나씩 시키고, 이유를 다시 묻는 습관**만 들이면 돼요.
